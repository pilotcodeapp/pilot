Pilot — project continuity doc

Last updated: March 20, 2026 (evening session)

What Pilot is

A local web app that acts as a bridge between Claude Chat and Claude Code. It gives Claude direct access to execute terminal commands while keeping the experience conversational — like claude.ai but with execution capabilities. Runs entirely on the user's existing Claude Pro/Max subscription with no separate API costs.

Current state

Working v2 running locally. Served via built frontend at localhost:3001. Phase 2 (Electron) is implemented — app runs as a native Mac desktop window via npm start or can still be used in browser. Core features:

  • React frontend (Vite) — built to dist/ and served by Express at localhost:3001
  • Node/Express backend with WebSockets at localhost:3001
  • PTY-based Claude Code integration (node-pty) — runs interactively, no API key needed
  • Auto-approves all actions (runs with --dangerously-skip-permissions); action prompt cards removed as they were just noise
  • Markdown rendering: bold, italic, lists, headings, inline code, fenced code blocks with copy button
  • Syntax highlighting in code blocks — highlight.js with support for JS/TS/JSX/TSX, Python, Bash, JSON, CSS, HTML, SQL, YAML, Rust, Go + auto-detection
  • Streaming text — Claude's responses render incrementally (word-by-word) like claude.ai; text coalesces into a single bubble per response (no fragmentation across tool calls)
  • User messages right-aligned (blue bubble), Claude messages left-aligned — clear visual separation
  • Auto-expanding textarea input
  • File attachment support — files uploaded via WebSocket, saved to /tmp/pilot-uploads/, path passed to Claude so it can read the file; images staged as pending with thumbnail preview
  • Screenshot paste support — Cmd+V pastes clipboard images; shows preview chip above input; user can add a message or send directly; Claude reads the image from disk (multimodal)
  • New chat / session reset
  • Session persistence — conversations saved to backend file store (~/.pilot/sessions/) as individual JSON files per session; shared across all devices hitting the same server; migrated from localStorage on first load; API: GET /sessions, PUT /sessions/:id, DELETE /sessions/:id
  • Session history shows project name label + supports double-click to rename
  • Project directory dropdown — reads from ~/.claude/projects + scans home for project markers (including one level deep for monorepo-style projects)
  • Multi-turn session continuity — uses --resume + --continue so Claude remembers prior turns
  • Expandable activity log — click "Show N steps" while Claude is working to see full timeline of actions; accumulates across turns within a session
  • Smart auto-scroll — only scrolls to bottom when user is near the bottom; does not yank user back down when reading earlier messages
  • Thinking indicator — stays visible the entire time Claude is working (only clears on session end, not between tool calls)
  • File tree panel (right sidebar) — shows full project directory structure with folder icons and collapsible folders; file names left-aligned with status badges (read/created/edited) right-aligned; auto-refreshes after each Claude turn; shows common dotfiles; supports up to 2000 files with truncation notice; badges persist via localStorage
  • Integrations section in file panel — shows Git, GitHub, Node/npm, Vercel, VS Code, Docker with latest action badges; detects all common git subcommands (add, init, remote, clone, tag, reset, etc.)
  • Preview panel — tabbed alongside file tree; iframe loads localhost URLs (auto-detected from Claude output) or renders previewable files (HTML, MD, images, PDF) directly when Claude creates/edits them; component sandbox for JSX/TSX files (React 18 + Babel standalone, renders default export live); subtle URL bar with refresh button; auto-refreshes iframe when any source file is edited; proxied through backend to avoid cross-origin issues
  • Dev server management — auto-detects and auto-starts npm run dev/npm run start when a project is selected; stop button available; auto-detects port from stdout and points preview iframe at it; cleans up on project switch or exit
  • Context meter — progress bar at bottom of left sidebar showing context window usage; reads from per-turn assistant event usage (not aggregated); shows compaction warning when Claude auto-compacts
  • Error handling — connection lost overlay, error messages in chat for Claude failures/unexpected exits, spawn failure detection
  • Quick action buttons — "Open localhost:XXXX" and "file.js in VS Code" appear above input; scoped to latest turn only (reset each message)
  • Resizable sidebars — drag-to-resize on both left sidebar and right files panel (Outlook-style)
  • Full-width layout — app stretches edge-to-edge across browser window
  • Portable — no hardcoded user paths; home directory fetched from backend /config endpoint; Claude binary path resolved dynamically
  • Electron desktop app — native Mac window with hidden title bar + traffic lights; starts Express server internally; npm start or npm run dev to launch; npm run build:mac to build DMG
  • Native folder picker — Electron's dialog.showOpenDialog exposed via IPC; frontend detects Electron environment and shows "Browse…" button for project selection; custom paths remembered in localStorage
  • Cloudflare Tunnel integration — persistent remote access via Cloudflare tunnels; two modes: token-based (user pastes admin-generated token) and persistent (admin's own tunnel with Cloudflare account); quick tunnels removed; auto-starts on server boot when configured (autoStart: true in ~/.pilot/tunnel.json); guided setup wizard triggered by toggle; domain: claudepilot.us; admin's tunnel at app.claudepilot.us
  • Remote authentication — password-protected access for remote connections; scrypt-hashed passwords in ~/.pilot/auth.json; session cookies (HttpOnly, Secure, SameSite=strict, 30-day); local access (localhost) always open; login screen component for unauthenticated remote users; WebSocket auth via cookie check; API endpoints return 401 for unauthenticated remote requests
  • Self-service registration — public page at /register; user picks username, backend auto-creates Cloudflare tunnel + DNS + returns token; rate-limited (3/hour/IP); requires admin config at ~/.pilot/admin.json (Cloudflare API token + account/zone IDs)
  • Admin CLI (scripts/pilot-admin) — bash script for managing user tunnels; commands: setup, add-user <name>, remove-user <name>, list-users; creates tunnels via Cloudflare API, configures remote ingress, generates tokens
  • Mobile enhancements — PWA manifest + icons (Add to Home Screen); haptic feedback on send; swipe-to-open sidebar (right swipe from left edge); 100dvh fix; code block scroll indicator (gradient fade); sticky input above iOS keyboard (visualViewport API); pull-to-refresh; long-press to copy code blocks; compact activity log on mobile (120px max-height + inline current step); file tree + preview via bottom sheet (FAB button, tabs); preview auto-open only for newly created files
  • WebSocket protocol upgrade — ws:// auto-upgrades to wss:// when page loaded over HTTPS (required for Cloudflare tunnel)
  • Resilient port binding — Electron tries ports 3001–3010 on startup; no more EADDRINUSE crashes
  • Pilot-aware system prompt — Claude knows it's inside Pilot via --append-system-prompt; instructs Claude to reference UI features (preview panel, file tree), mention component auto-preview, and points to the continuity doc
  • App icon — compass rose with < /> angle brackets at center; warm Claude-inspired gradient (peach/amber); SVG source at icon-compass-rose.svg; assets/icon.icns for Electron; favicon and sidebar logo in frontend
  • Session context recovery — resuming a saved session prepends conversation history to the first prompt so Claude has context even without a live --resume session
  • Onboarding / setup screen — on first launch, /health endpoint checks Node.js availability, Claude Code installation, and authentication; if anything is missing, SetupScreen component guides user through prerequisites (Node.js install options: nodejs.org / Homebrew / nvm), one-click Claude Code install via POST /setup/install-claude, and authentication instructions
  • pilotctl CLI — bash script for managing Pilot as a macOS LaunchAgent; commands: start, stop, restart, status, logs, errors, build (rebuilds frontend + restarts); targets ~/Library/LaunchAgents/com.pilot.app.plist

Architecture (post-refactor)

Three layers: Electron shell → Node/Express backend → React frontend. Frontend was refactored from a single 1400-line App.jsx into focused components:

  • ~/pilot/electron/main.js — Electron entry: BrowserWindow (1400×900, min 900×600), hidden title bar with traffic lights, IPC handler for native folder picker with security-scoped bookmarks (MAS), port-finding (3001–3010), starts Express server on launch, cleanup on quit
  • ~/pilot/electron/preload.js — context bridge exposing electronAPI.openFolder() to renderer
  • ~/pilot/electron/entitlements.mas.plist — MAS sandbox entitlements (sandbox, network, file access, bookmarks, child process inherit)
  • ~/pilot/electron/entitlements.mas.inherit.plist — inherited entitlements for child processes (Claude Code)
  • ~/pilot/pilotctl — bash CLI for LaunchAgent management (start/stop/restart/status/logs/errors/build)
  • ~/pilot/frontend/src/App.jsx — orchestrator: state, WebSocket, effects, layout wiring (~710 lines)
  • ~/pilot/frontend/src/utils.js — constants, helpers (describeAction, extractFileActivity, extractQuickActions, formatInline, session storage)
  • ~/pilot/frontend/src/components/ChatPanel.jsx — messages list, input bar, quick actions, thinking indicator, connection overlay
  • ~/pilot/frontend/src/components/Sidebar.jsx — project selector, session list, context meter, new chat button
  • ~/pilot/frontend/src/components/RightPanel.jsx — tabs wrapper (Files/Preview)
  • ~/pilot/frontend/src/components/FileTree.jsx — directory tree rendering + integrations section
  • ~/pilot/frontend/src/components/PreviewPanel.jsx — URL bar, dev server controls, iframe
  • ~/pilot/frontend/src/components/Markdown.jsx — markdown-to-JSX rendering
  • ~/pilot/frontend/src/components/CodeBlock.jsx — syntax highlighting + copy button (hljs setup)
  • ~/pilot/frontend/src/components/SetupScreen.jsx — onboarding flow: checks Node.js + Claude Code + auth status via /health; guides install + auth; states: default → no_node / installing / auth_needed / error
  • ~/pilot/frontend/src/components/LoginScreen.jsx — password login for remote access; shown when /auth/status returns login_required
  • ~/pilot/frontend/src/components/RegisterPage.jsx — self-service registration; username input, calls /api/register, shows token + setup instructions
  • ~/pilot/frontend/src/components/AccountScreen.jsx — first-launch account registration (name + email); sends to central server
  • ~/pilot/scripts/pilot-admin — bash CLI for managing user tunnels via Cloudflare API; config at ~/.pilot/admin.json
  • ~/pilot/frontend/src/App.css — all styles
  • ~/pilot/backend/server.js — WebSocket server + dual-mode Claude spawn (node-pty with fallback to child_process.spawn for MAS sandbox) + endpoints: /config, /health (Node/Claude/auth checks), /setup/install-claude, /open, /projects, /filetree, /preview-file, /preview-proxy, /dev-server/detect + dev server lifecycle management via WebSocket; exports { server, app, wss, PORT } so Electron can require and control it (only auto-listens when run directly)
  • Claude Code binary path — resolved dynamically from $CLAUDE_PATH env var or ~/.local/bin/claude
  • ~/pilot/worker/index.js + worker/wrangler.toml — Cloudflare Worker for always-on account registration + heartbeat; KV-backed; deployed to claudepilot.us/api/*
  • ~/pilot/landing/index.html + landing/favicon.svg — static landing page; deployed to Cloudflare Pages (pilot-landing project) at claudepilot.us
  • ~/pilot/README.md — project README with download link, run-from-source, features, architecture, scripts

How to start Pilot

  • Electron (desktop app): cd ~/pilot && npm start — opens native window, starts server automatically
  • Browser mode: node ~/pilot/backend/server.js then open http://localhost:3001
  • LaunchAgent (background): pilotctl start — runs as macOS service; also stop, restart, status, logs, errors, build
  • Build DMG: npm run build:mac — builds frontend then packages with electron-builder (DMG target, category: developer-tools)
  • After frontend code changes, rebuild: cd ~/pilot/frontend && npm run build then refresh browser (or pilotctl build)
  • After backend changes, restart: kill and re-run node ~/pilot/backend/server.js (or pilotctl restart)
  • Node version must be v20 — if wrong run nvm use 20 first

Known issues

  • node-pty requires Node v20 — does not work on Node v25
  • Preview panel iframe may show "refused to connect" for some localhost services depending on their CORS/CSP headers — the proxy strips x-frame-options but some apps set additional restrictions
  • Auth sessions are in-memory — server restart clears all remote sessions (users need to re-login); could move to file-based if this becomes an issue
  • Registration page (/register) requires ~/.pilot/admin.json to be configured via pilot-admin setup first
